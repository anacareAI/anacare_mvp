## What Was Built

### Project Structure (WAT Framework)
```
tools/          12 Python tools covering the full pipeline
workflows/      4 SOPs: MRF ingestion, OOP calc, quality scoring, ranking API
api/            FastAPI application (main.py, db.py)
tests/          38 unit tests (all passing)
Makefile        Developer convenience targets
```

### Tools
- fetch_mrf_index.py    — Download + validate Aetna MRF index JSON
- stream_mrf_file.py    — ijson streaming, CPT filter, .jsonl output
- parse_rates.py        — Normalize + deduplicate rate records
- resolve_npi.py        — CMS NPI Registry lookup with resume support
- load_rates_db.py      — Upsert rates + providers to PostgreSQL
- schema.sql            — Full DB schema (providers, rates, quality_scores, ingestion_log)
- compute_oop.py        — Deductible/coinsurance/OOP-max formula + range mode
- fetch_quality_cms.py  — CMS Provider Data Catalog API
- score_quality.py      — Weighted quality index, 40% coverage suppression
- map_cpt.py            — Claude API NL->CPT mapping
- rank_providers.py     — Percentile-rank composite scoring
- format_output.py      — API response serialization + disclaimer
- geocode.py            — Haversine distance + zip radius filter
- download_zip_data.py  — Census ZCTA centroid downloader (run once)

### API Endpoints
- POST /oauth/token         — Sandbox token
- POST /v1/cpt-lookup       — NL procedure -> CPT candidates via Claude
- POST /v1/rank-providers   — Core ranking query
- GET  /v1/providers/{npi}  — Single provider detail
- GET  /v1/health           — Health check
